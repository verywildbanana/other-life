## 근본 원인 (2026-06-10 발견)

**SSR(서버사이드 렌더링) 미적용**이었다.

- 기존: `FeedView.tsx`가 `useState([])` → HTML이 빈 상태로 전달 → Googlebot이 JS 실행 없이 보면 **빈 페이지**
- Googlebot은 저권위 도메인의 JS 렌더링을 후순위로 처리 → 콘텐츠 없음 판단 → 인덱스 제거
- 결과: GSC에 등록해도 "처음 크롤링 임시 노출" 후 품질 평가에서 제거되는 패턴 3개월 반복

## 수정 내용 (2026-06-10 배포)

### 1. `app/page.tsx` — 루트 서버사이드 리디렉션
```tsx
// 변경 전: 클라이언트 JS로 router.replace()
// 변경 후: 서버에서 Accept-Language 헤더 감지 후 redirect()
import { headers } from 'next/headers'
import { redirect } from 'next/navigation'

export default async function RootPage() {
  const headersList = await headers()
  const acceptLang = headersList.get('accept-language') ?? ''
  let lang = 'en'
  if (acceptLang.toLowerCase().includes('ko')) lang = 'ko'
  else if (acceptLang.toLowerCase().includes('ja')) lang = 'ja'
  redirect(`/p/wealthy_single_30s?lang=${lang}`)
}
```

### 2. `app/p/[persona_id]/page.tsx` — SSR 피드 데이터 fetch
```tsx
// 변경 전: feed={null} 로 FeedView에 전달
// 변경 후: 서버에서 getPaginatedFeed() 호출 후 초기 데이터 전달
const [persona, allPersonas, initialFeed] = await Promise.all([
  loadPersonaAsync(persona_id),
  listAllPersonas(),
  getPaginatedFeed(persona_id, 0, 20, true).catch(() => null),
])
return <FeedView feed={initialFeed} persona={persona} allPersonas={allPersonas} />
```

### 3. `app/p/[persona_id]/FeedView.tsx` — 초기 상태에 SSR 데이터 사용
```tsx
// 변경 전:
const [videos, setVideos] = useState<Video[]>([])
const [isInitialLoading, setIsInitialLoading] = useState(true)

// 변경 후:
const [videos, setVideos] = useState<Video[]>(() => feed?.videos ?? [])
const [isInitialLoading, setIsInitialLoading] = useState(!(feed?.videos?.length))
```

**효과**: HTML에 영상 제목 20개가 포함된 채로 서버에서 전달 → Googlebot이 JS 없이 콘텐츠 읽기 가능

## 참고: "처음엔 나왔다가 사라진" 이유

GSC에 도메인 등록 직후 Google이 즉시 크롤링 → 품질 평가 없이 일시적으로 `site:` 검색에 표시.
이후 품질 평가에서 "빈 페이지(JS 미실행 시 콘텐츠 없음)" 판단 → 인덱스에서 제거.
이 패턴이 3개월간 반복됐음. **SSR 수정 전까지는 어떤 요청을 해도 의미 없었던 구조.**
